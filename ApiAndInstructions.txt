📌 User Module (/api/users)
Method	Endpoint	Description	Request Body	Response	Access
POST	/users/create	Create a new user (after Firebase login)	{ uid, name, mobile, address, role? (default "customer"), organizationId? }	{ success: true, data: User }	Firebase Auth
GET	/users/uid/:uid	Get user by Firebase UID	–	{ success: true, data: User }	Self or Admin
PATCH	/users/profile	Update own profile	{ name?, mobile?, address? }	{ success: true, data: User }	Self
PATCH	/users/fcm-token	Update FCM token	{ fcmToken }	{ success: true, data: User }	Self
PATCH	/users/role	Update user role (admin only)	{ uid, role }	{ success: true, data: User }	Admin
PATCH	/users/status	Update user status (admin only)	{ uid, status }	{ success: true, data: User }	Admin
GET	/users	List all users (with optional filters)	Query: ?role=&status=&organizationId=&limit=&skip=	{ success: true, users: [], pagination }	Admin
PATCH	/users/role/:uid	Update user role by UID in URL	{ role }	{ success: true, data: User }	Admin

📌 Important Notes
Date Format: All dates should be sent as ISO 8601 strings (e.g., 2025-03-03T10:30:00.000Z).

Pagination: Most list endpoints support limit and skip query parameters.

Error Responses: Always contain { success: false, message: "error description" }.

File Uploads: For endpoints accepting picUrl, you can send a file using multipart/form-data with field name picUrl. The backend will upload to Cloudinary and store the URL.
